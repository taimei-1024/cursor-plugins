---

## 评审结果判定标准

| 结果 | 条件 |
|------|------|
| **通过** | 无严重问题，无需要澄清的问题 |
| **有条件通过** | 无严重问题，但有需要澄清的问题或重要改进项 |
| **需要修改** | 存在严重问题，需要修改后重新评审 |

---

## 文件命名规则

格式：`{JiraKey}-review-{YYYYMMDD-HHmmss}.md`

示例：
- `PVS-123-review-20240115-143000.md`
- `PVS-456-review-20240120-091530.md`

---

## 存储位置

评审报告存储在项目的 `reviews/` 目录下。

目录结构：
```
pvs-report/
├── reviews/
│   ├── PVS-123-review-20240115-143000.md
│   ├── PVS-456-review-20240116-100000.md
│   └── ...
└── ...
```

## 钉钉通知消息模板

评审完成后发送钉钉通知，使用以下 Markdown 格式：

```markdown
## 技术设计评审完成

**基本信息**
- Jira Key: {JiraKey}
- 需求标题: {需求标题}
- 需求类型: {新功能开发/功能迭代/技术优化}
- 评审时间: {YYYY-MM-DD HH:mm}

**评审结果: {通过/有条件通过/需要修改}**

| 统计 | 数量 |
|------|------|
| 通过项 | {n}/39 |
| 改进项 | {n} |
| 澄清项 | {n} |

**主要风险点**
{列出 1-3 个主要风险点，如无则写"无"}

> 详细报告已保存至 reviews/{文件名}
```

### 通知示例

```markdown
## 技术设计评审完成

**基本信息**
- Jira Key: PVS-1234
- 需求标题: ICSR 征集报告比对功能
- 需求类型: 功能迭代
- 评审时间: 2024-01-15 14:30

**评审结果: 有条件通过**

| 统计 | 数量 |
|------|------|
| 通过项 | 34/39 |
| 改进项 | 4 |
| 澄清项 | 1 |

**主要风险点**
1. 发起比对接口缺少幂等性设计
2. tm_compare_result 表缺少必要索引
3. 字段配置服务故障时缺少降级方案

> 详细报告已保存至 reviews/PVS-1234-review-20240115-143000.md
```
