# 技术设计文档模板索引

本目录包含三个技术设计文档模板，分别适用于不同的设计场景。

---

## 模板选择指南

| 场景 | 推荐模板 | 文件 |
|------|----------|------|
| 新功能开发、系统架构设计 | 模板一：总体设计 | [standards/general-design-template.md](standards/general-design-template.md) |
| 单功能开发、接口开发、小范围迭代 | 模板二：后端详细设计 | [standards/backend-detail-design-template.md](standards/backend-detail-design-template.md) |
| 前端页面开发、交互设计、组件开发 | 模板三：前端详细设计 | [standards/frontend-detail-design-template.md](standards/frontend-detail-design-template.md) |
| 技术优化、重构 | 模板一或模板二（根据规模） | - |

---

## 使用方式

### 文档大纲生成

**触发词**：生成大纲、文档骨架、设计模板

**输入**：需求描述 / Jira Key

**执行步骤**：
1. 确认需求类型和规模
2. 选择合适的模板
3. 按模板章节生成文档大纲
4. 标注必写/选写章节

---

## 模板章节说明

### 总体设计模板

**必写章节**（标注 *）：
- 1.1 系统/服务概述
- 1.2 文档概述
- 3.1 系统/组件架构
- 3.2.1 关键算法和技术实现或选型
- 3.3.2 系统流程图
- 3.4.1 层次结构
- 3.5.1 性能及响应时间
- 3.5.5 资源使用

**完整结构**：1.概述 → 2.服务依赖 → 3.概要设计 → 4.参考文档

### 后端详细设计模板

**必写章节**（标注 *）：
- 1. 功能描述
- 3.1 设计思路
- 3.3 ER 设计（有数据变更时）
- 3.4 接口设计（有接口变更时）

**完整结构**：1.功能描述 → 2.需求 → 3.技术设计 → 4.影响范围 → 5.参考文档

### 前端详细设计模板

**必写内容**：
- 概述（Jira 地址、排期）
- 相关文档（需求、交互、视觉、后端、接口、测试）
- 设计思路（功能说明、流程说明、详细设计、外部依赖）
- 技术分析 Checklist（10 项检查）

**完整结构**：1.概述 → 2.相关文档 → 3.总体设计 → 4.技术分析 Checklist → 5.测试数据 → 6.测试重点

## 技能调用提示

在生成文档大纲时，提示以下技能可辅助完善内容：

| 章节 | 可用技能 |
|------|----------|
| 流程图/时序图/架构图 | `flowchart` |
| 表结构设计 | `table-spec` |
| 接口明细 | `tech-design-review` API 契约表生成 |
| 方案对比 | `tech-design-review` 架构方案对比 |
| 安全自查 | `tech-design-review` 安全漏洞自查 |
