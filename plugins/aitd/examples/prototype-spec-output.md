## 产品原型说明书

### 一、页面概述

**页面名称**: 安全报告创建
**业务场景**: 临床试验安全员为患者创建不良事件报告
**用户角色**: 安全员 (Safety Officer)

---

### 二、区域划分

| 序号 | 区域名称 | 位置 | 功能描述 |
|-----|---------|------|---------|
| 1 | 患者信息区 | 顶部 | 选择/搜索要报告的患者 |
| 2 | 不良事件区 | 中上部 | 记录不良事件详情 |
| 3 | 药品信息区 | 中部 | 记录可疑药品信息 |
| 4 | 报告类型区 | 中下部 | 选择报告类型和严重程度 |
| 5 | 操作按钮区 | 底部 | 保存、提交、取消 |

---

### 三、字段详情

#### 3.1 患者信息区

| 字段名称 | 类型 | 是否必填 | 说明/逻辑 |
|---------|------|---------|----------|
| 患者 | 搜索选择 | 是 | 支持按姓名/筛选号搜索，选择后带出患者基本信息 |
| 筛选号 | 只读 | - | 选择患者后自动填充，如 "P001-001" |
| 项目名称 | 只读 | - | 选择患者后自动填充 |

#### 3.2 不良事件区

| 字段名称 | 类型 | 是否必填 | 说明/逻辑 |
|---------|------|---------|----------|
| 事件名称 | 下拉选择 | 是 | MedDRA 词典，支持搜索 |
| 事件描述 | 多行文本 | 条件必填 | 选择"其他"时必填，至少 20 字 |
| 开始日期 | 日期 | 是 | 默认今天，不能晚于今天 |
| 结束日期 | 日期 | 否 | 开始日期之后，可空表示持续中 |
| 严重程度 | 单选 | 是 | 轻度/中度/重度/致死 |

#### 3.3 药品信息区

| 字段名称 | 类型 | 是否必填 | 说明/逻辑 |
|---------|------|---------|----------|
| 怀疑药品 | 下拉选择 | 是 | 项目药品清单，支持搜索 |
| 药品名称 | 只读 | - | 选择后自动填充 |
| 给药途径 | 下拉选择 | 是 | 口服/静脉/肌肉/皮下/其他 |
| 用法用量 | 文本 | 是 | 如 "每日一次，每次 10mg" |
| 开始用药日期 | 日期 | 是 | 默认今天，不能晚于今天 |
| 停药日期 | 日期 | 否 | 开始用药日期之后 |

#### 3.4 报告类型区

| 字段名称 | 类型 | 是否必填 | 说明/逻辑 |
|---------|------|---------|----------|
| 报告类型 | 单选 | 是 | 初次报告/随访报告/合并报告 |
| 严重不良事件 | 是/否 | 是 | 选择"是"时触发后续流程 |
| 因果关系 | 单选 | 条件必填 | 严重不良事件时必填：肯定/很可能/可能/可能无关/无关 |

#### 3.5 操作按钮区

| 按钮 | 触发条件 | 功能 |
|------|---------|------|
| 保存草稿 | 所有必填项填写完成 | 保存为草稿状态，可继续编辑 |
| 提交报告 | 所有必填项填写完成 + 校验通过 | 提交报告，状态变为"待审核" |
| 取消 | - | 返回列表页，未保存的数据丢失提示 |

---

### 四、交互逻辑

#### 4.1 显隐联动

- 选择"严重不良事件 = 是"时，显示"因果关系"字段
- "报告类型 = 随访报告"时，显示"原报告编号"字段
- "事件名称 = 其他"时，"事件描述"变为必填且标注红色星号

#### 4.2 校验规则

| 字段 | 校验规则 |
|-----|---------|
| 开始日期 | 不能晚于当前日期 |
| 结束日期 | 如有，必须晚于开始日期 |
| 停药日期 | 如有，必须晚于开始用药日期 |
| 用法用量 | 不能为纯空格，至少 2 个字符 |

#### 4.3 输入限制

- 事件描述：最多 500 字
- 用法用量：最多 100 字
- 所有日期选择器不可选择未来日期

#### 4.4 下拉选项

- **事件名称**: 调用 MedDRA 词典接口，支持搜索
- **怀疑药品**: 调用项目药品清单接口
- **给药途径**: 固定选项（口服/静脉/肌肉/皮下/其他）

### 六、备注说明

1. **标红规则**: 所有必填项未填写时，标签显示为红色星号 (*)
2. **保存提示**: 点击"保存草稿"成功后，显示 Toast 提示"保存成功"
3. **提交校验**: 点击"提交报告"时，先校验必填项，如有未填项则聚焦到第一个未填项
4. **数据来源**: 患者信息、药品清单均从后端接口获取
