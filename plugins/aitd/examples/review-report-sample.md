## 四、改进建议

### 4.1 架构层面

1. **拆分 WorkflowConfigService**:
   - `WorkflowConfigReader`: 负责配置读取和解析
   - `WorkflowConfigValidator`: 负责配置校验
   - `WorkflowExecutor`: 负责流程执行

2. **明确服务边界**:
   - WorkflowConfigService: 配置管理
   - ProcessEngine: 流程执行
   - 两者通过接口解耦

### 4.2 数据层面

3. **增加联合索引**:
   ```sql
   ALTER TABLE tm_workflow_config
   ADD INDEX idx_tenant_project (tenant_id, project_id);
   ```

### 4.3 接口层面

4. **增加幂等性处理**:
   - 请求头增加 `X-Workflow-Exec-Id`
   - 使用 Redis 去重，TTL 10 分钟
