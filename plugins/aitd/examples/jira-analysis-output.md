## 详细描述

当前登录页面存在暴力破解风险，需要增加图形验证码功能以提高系统安全性。

### 业务背景

近期发现多个账户存在异常登录尝试，可能是暴力破解攻击。为保障用户账户安全，需要在登录流程中增加人机验证环节。

### 需求目标

1. 防止自动化脚本暴力破解
2. 提升用户账户安全性
3. 对正常用户影响最小化

## 图片分析

### 附件 1: login-mockup.png

**整体布局**:
- 顶部: 公司 Logo + "用户登录" 标题
- 中部: 表单区域（用户名、密码、验证码输入框）
- 底部: "登录" 按钮 + "忘记密码" 链接

**新增组件**:
- **验证码输入框**: 位于密码输入框下方，宽度 120px
- **验证码图片**: 显示 4 位字母数字组合，支持点击刷新
- **刷新图标**: 位于验证码图片右侧，点击重新获取验证码

**字段详情**:

| 字段名称 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| username | 文本 | 是 | 用户名/邮箱/手机号 |
| password | 密码 | 是 | 登录密码 |
| captcha | 文本 | 是 | 图形验证码，4 位字符 |

---

## 验收标准

1. 验证码为 4 位字母数字组合（区分大小写）
2. 验证码有效期为 60 秒
3. 输入错误 3 次后锁定账户 5 分钟
4. 验证码图片支持点击刷新
5. 验证码字符清晰可辨，无明显歧义
6. 登录失败后验证码自动刷新

---

## 技术考虑

### 性能要求

- 验证码生成接口 RT < 100ms
- 验证码图片大小 < 10KB

### 安全要求

- 验证码服务防刷机制
- 验证码存储使用 Redis，过期时间 60 秒
- 登录失败次数基于 IP + 用户名双重计数

## 依赖接口

| 接口 | 方法 | 说明 |

| /api/captcha/verify | POST | 校验验证码 |
| /api/auth/login | POST | 用户登录 |
