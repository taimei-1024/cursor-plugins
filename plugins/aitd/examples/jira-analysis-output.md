### 二、新增"导出记录"功能

**位置**：数据导出一级菜单下

**交互样式**：同文件中心

**功能特性**：
- 文件及标签页标题：导出记录 (`export_records`)
- 包含分页器
- 包含搜索（按文件名称）

**表格列**：

| 列名 | 说明 |
|------|------|
| 文件名称 | - |
| 发送至 | 本地/Email/SFTP |
| 状态 | 已完成/失败 |
| 创建人 | 生成定时任务的人 |
| 创建时间 | 定时任务生成的时间 |

**操作**：
- 查看配置按钮：点击可查看当时的快照，不允许编辑

## 结构化需求摘要

1. **国际化文案统一**：统一所有导出操作的提示文案为 `prompt_export_success_file_center`

2. **模板文件排除**：所有模板类文件不再进入下载中心

3. **新增导出记录页面**：
   - 位置：数据导出菜单下
   - 参考：文件中心交互
   - 功能：分页、搜索、查看配置快照
   - 字段：文件名称、发送目标、状态、创建人、创建时间

4. **影响范围**：16个功能模块的导出/下载行为

## 技术考虑

### 数据模型

**导出记录表** (export_records):

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 主键 |
| project_id | bigint | 项目ID |
| file_name | varchar(255) | 文件名称 |
| send_target | varchar(50) | 发送目标：LOCAL/EMAIL/SFTP |
| status | varchar(20) | 状态：SUCCESS/FAILED |
| creator_id | bigint | 创建人ID |
| created_time | datetime | 创建时间 |
| config_snapshot | text | 配置快照（JSON） |

### 接口设计

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/export-records | GET | 查询导出记录列表 |
| /api/export-records/{id}/config | GET | 查看配置快照 |
| /api/export/notify-center | POST | 发送到下载中心 |

### 国际化 Key

需要新增/修改的国际化 key：
- `export_records` - 导出记录
- `prompt_export_success_file_center` - 文件已发送至下载中心
